# Система заказов

Заказы - неотъемлемая игровая механика в данной игре. Ниже описана как реализована данная механика и как можно взаимодействовать с заказами, клиентами и игроком.

## Менеджер заказов

Класс `OrdersManager` позволяет хранить, удалять, добавлять, получать заказы.

Заказ имеет следующую структуру данных:
```cpp
struct Order {
	unsigned int id;
	std::string title;
	const std::string icon;
	float wait;
	bool cookeed = false;
};
```

Данные для формирования заказа получаются из другого класса - [Items](../../src/mechanics/items.hpp).

- `id` - идентификатор заказа
- `title` - название заказа
- `icon` - иконка заказанного предмета
- `wait` - время ожидания клиента
- `cookeed` - отслеживание состояние приготовления заказа


### Методы

Добавить заказ.

```cpp
void addOrder(const std::string name, const Order order);
```

Получить данные структуры `Order`.

```cpp
Order* getOrder(const std::string name);
```

Удаляет указанный рецепт из контейнера.

```cpp
void removeOrder(const std::string name);
```

## Клиент

Клиенты - [NPC](./entities.md#npc), которые приходят в ресторан игрока и заказывают случайное блюдо из доступного в меню. 

Учитываются следующие условия для формирования заказа:
1. Изучен ли рецепт блюда
2. Есть ли он в меню 
3. Есть ли ингредиенты для его приготовления

Если какой-либо условие ложь - происходит перегенерация заказов. Лимит перегенерации - 10. Если клиент не может сформировать заказ (пустое меню, нет ингредиентов для приготовления) - он уходит, снижая [рейтинг](./rating.md) ресторана.

## Отображение заказов

Когда появляеться заказ - влевом верхнем углу появиться заметка, в котором будет отображаться:

- Иконка блюда 
- Название блюда
- Прогресс-бар ожидания клиента

Если ожидание клиента достигнет нуля - заказ будет завершен, клиент уйдет, а рейтинг ресторана снизиться на 0.25.
